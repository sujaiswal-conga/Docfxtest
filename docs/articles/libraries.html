Composer DocGen Libraries

Composer NextGen is built on Conga Platform and the core domain logic (document processing, data management etc.) is broken down into set of libraries.

Libraries

  • Conga.CoreApps.DocGen.SDK - A C# library that helps any .NET application to easily create and execute document generation workflows. The library under the hood send the requests to NextGen services to run the workflows.

  • Conga.CoreApps.DocGen.Models - Library that contains the re-usable model classes for document generation that is used by both the SDK and the microservices.

  • Conga.CoreApps.DocGen.DataSet - Library that extends the built-in DataSet (SmartSet) and adds additional capabilities like retrieving data using XPath like syntax, binds custom metadata to tables and columns etc.

  • Conga.CoreApps.DocGen.Base - Provides abstractions for all document libraries.

  • Conga.CoreApps.DocGen.Data - Provides common models and abstractions for querying any platform like RLS, Salesforce etc.

  • Conga.CoreApps.DocGen.RLS - Provides common models and abstractions for querying from RLS platform.

  • Conga.CoreApps.DocGen.Salesforce - Provides common models and abstractions for querying from Salesforce platform.

-Conga.CoreApps.DocGen.Word - House of all class libraries for processing MS Word documents. Contains Merge Engines, template transformers, document processing actions etc.

  • Conga.CoreApps.DocGen.Pdf - Contains classes to work with PDF documents.

  • Conga.CoreApps.DocGen.Messages - Contains re-usable message classes that is used by Microservices to communicate for document generation workflows.

  • Conga.Dragon - Framework for distributed programming. Dragon helps to simplify programming when you have a distributed system made-up of microservices that communicates each other through messaging.
