Conga.Dragon

The setup modules simplifies to configure Dragon with pre-defined modules like messaging, caching with concrete implementations.

ConfigureDragon extension method

The ConfigureDragon is an extension to IServiceCollection and you need to invoke this method usually in the ConfigureServices method of Startup. Below is the signature of the method.

public static IServiceCollection ConfigureDragon(this IServiceCollection services, IConfiguration configuration, DragonConfig? dragonConfig = null)

DragonConfig class.

This class helps to pass configuration parameters while configuring dragon.

public class DragonConfig
{
	public bool ConfigureSubscriber { get; set; }
}

Dragon class

This is a singleton class that helps to bootstrap the Dragon framework. It has two methods Fly and Down. The Fly method DragonStartupParams and which should be called to start the framework. The Down method does the tear down work.

Methods

void Fly(DragonStartupParams dragonSetup)

void Down()

Usually you don't have to reference this module directly in web or worker and you should go with the extensions. This module is usually referenced from your infrastructure project.
