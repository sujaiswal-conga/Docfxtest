Conga.Dragon.Domain

This module contains project that contains artifacts to simplify Domain Driven Development. It also provides to write commands and queries (CQRS) easily and attach custom behaviors to them.

IEntity

Any component that is an identity should implement this interface. You don't have to directly use this interface lot of times and instead you should refer BaseEntity which implements this interface.

BaseEntity

Any class that is an entity should derive from this abstract class. An entity is something that has an identity and can be persisted to database. The class contains generic properties like Id, Name, CreatedDate, LastModifiedDate etc.

IAggregateRoot

Represents an aggregate root. An aggregate root is a class which works as an entry point to our aggregate. All business operations should go through the root. This way, the aggregate root can take care of keeping the aggregate in a consistent state. The root is what takes cares of all our business invariants. Instead of using this interface please use BaseAggregateRoot. Refer: https://martinfowler.com/bliki/DDD_Aggregate.html.

BaseAggregateRoot

The abstract base class for all aggregate roots.

IValueObject

Represents a value object. A value object is a design pattern in which an object is made to represent something simple, like currencies or dates. For more details refer https://martinfowler.com/bliki/ValueObject.html.

BaseCommand

The abstract base class for all commands that can be invoked in a busines domain or bounded context. This class implements IRequest of MediatR library.

BaseQuery

Represents a query that pulls and return data from the domain.

BaseCommandHandler

All command handlers in your domain should derive from this class. It implements MediatR's IRequestHandler. Takes care of validating all commands before invoking them.

ICommandBehavior

Represents a command that attaches custom behavior to a command or query at domain level. You can also use the CommandBehaviorAttribute to do the same for specific command(s) or query(s).

CommandBehaviorAttribute

Represents a class that affects the behavior of a command / query.

IPort

Represents a port through which an external system can access the domain or from domain we can interact with an external system. For more information please google "Ports & Adapters" or hexagonal architecture.

IDbRepository

Represents an external port that helps to interact with database ops.

IAppEventPublisher

Represents an outbound port that helps to publish domain events.

DomainEvent

An abstract class that represents an event published from the domain.
